## 1. 项目概述

### 1.1 产品定位

面向中大企业的**工程项目全生命周期管理平台**，覆盖从投标立项、计划编制、任务执行、文件交付到财务结算的全流程。

### 1.2 核心能力

| 能力域 | 说明 |
|-------|------|
| 项目台账 | 项目创建、复制、状态流转、成员/里程碑维护 |
| WBS 任务 | 树形任务分解、甘特图展示、前置依赖、进度追踪 |
| 文件管控 | 模板化文件清单、版本管理、角色级可见性控制、逾期预警 |
| 模板复用 | 计划/里程碑/成员/文件四类模板，一键装配到项目 |
| 权限体系 | RBAC 角色 + 数据范围 + 字段脱敏 + 文件可见性，四层复合权限 |
| 财务管理 | 合同金额、收付款计划、实际收款、开票记录 |
| 变更管理 | 项目变更请求的记录和追踪 |
| 操作审计 | 关键操作前后数据 diff 记录，支持按时间/部门/人员查询 |

## 2. 用户与角色

### 2.1 角色定义

| 角色 | 编码 | 数据范围 | 职责 |
|-----|------|---------|------|
| 系统管理员 | `admin` | 全部数据 | 系统配置、用户管理、权限分配 |
| 模板管理员 | `template_admin` | 全部数据 | 模板创建和维护 |
| 项目管理员 | `project_admin` | 部门及子级 | 本部门项目的全量管理 |
| 项目经理 | `project_manager` | 自有项目 | 名下项目的任务/文件/变更管理 |
| 任务管理 | `task_manager` | 参与项目 | 参与项目的任务查看与更新 |
| 普通用户 | `user` | 本人相关 | 个人待办、文件上传 |

### 2.2 权限模型

```
用户 → RBAC 角色 → 权限码集合 → 菜单可见性 + 操作许可
                → 数据范围   → 行级数据过滤
                → 字段权限   → 敏感字段脱敏
```

- **权限码数量**：60+，按树形组织（菜单 → 页面 → 按钮 → 字段）
- **数据范围 6 级**：全部 / 本部门 / 部门及子级 / 仅本人 / 项目成员 / PM 自有
- **字段脱敏**：19 个敏感字段（合同编号、客户名称、联系方式、财务数据等）

## 3. 认证与安全

### 3.1 登录认证

| ID | 需求 | 详情 |
|----|------|------|
| AUTH-01 | 用户名密码登录 | 使用 BCrypt 哈希验证，成功后签发 JWT，有效期 24h |
| AUTH-02 | 密码修改 | 需验证原密码，新密码 ≥ 6 位，修改后需重新登录 |
| AUTH-03 | Token 校验 | 所有 API（除登录和公开接口）需携带 `Authorization: Bearer` 头 |
| AUTH-04 | 文件下载认证 | 支持 `?access_token=` 查询参数传递 JWT（浏览器下载限制） |
| AUTH-05 | 自动证书 | 首次启动自动生成自签名 HTTPS 证书 |

### 3.2 权限控制

| ID | 需求 | 详情 |
|----|------|------|
| AUTH-06 | 动态权限策略 | `[RequirePermission("code")]` 特性，运行时解析，无需预注册 Policy |
| AUTH-07 | Admin 豁免 | 系统管理员自动跳过所有权限检查 |
| AUTH-08 | 菜单动态生成 | 根据用户权限码过滤菜单树，拥有子权限时自动展开父菜单 |
| AUTH-09 | 隐式权限继承 | `project:list` → 自动获得 `project:list:view` + `project:detail:view` |

## 4. 项目管理

### 4.1 项目基本信息

| ID | 需求 | 详情 |
|----|------|------|
| PRJ-01 | 创建项目 | 必填：项目编号、项目名称；可选：类型、客户、销售人员、项目经理、工期等 30+ 字段 |
| PRJ-02 | 项目编号唯一 | 系统自动校验，重复时拒绝创建 |
| PRJ-03 | 列表查询 | 分页 + 多条件筛选（编号/名称/类型/状态/PM/成员/责任人），含数据范围过滤 |
| PRJ-04 | 项目详情 | 含产品、成员、里程碑完整信息；**无字段权限时脱敏显示**（返回 null 或 `***`） |
| PRJ-05 | 编辑项目 | 仅**未激活**状态可修改基本信息；已激活项目锁定关键字段 |
| PRJ-06 | 删除项目 | 需 `project:delete` 权限 + 维护权限，物理删除 |
| PRJ-07 | 复制项目 | 全量复制（项目信息 + 成员 + 产品 + 里程碑 + 任务树 + 变更 + 财务），自动编号 `xxx-副本` |

### 4.2 项目状态流转

```
          ┌──────────┐
          │  0 未激活  │ ← 新建项目默认状态
          └────┬─────┘
               │ 激活
               ▼
       ┌──────────┐
  ┌───→│  1 进行中  │←─── 恢复 ───┐
  │    └──┬───┬───┘              │
  │       │   │                   │
  │  完成 │   │ 暂停              │
  │       │   ▼                   │
  │       │  ┌──────────┐        │
  │       │  │  3 已暂停  │────────┘
  │       │  └──────────┘
  │       ▼
  │  ┌──────────┐
  └──│  2 已完成  │ (可重新激活)
     └──────────┘
```

| ID | 需求 | 详情 |
|----|------|------|
| PRJ-08 | 激活 | 任意状态 → 1 |
| PRJ-09 | 完成 | → 2 |
| PRJ-10 | 暂停 | → 3 |
| PRJ-11 | 恢复 | 3 → 1，**非暂停状态拒绝恢复** |
| PRJ-12 | 反激活 | 1 → 0，**仅管理员和项目管理员可操作**，项目经理不可反激活 |

### 4.3 项目成员

### 4.4 项目里程碑

## 5. 任务管理（WBS + 甘特图）

### 5.1 任务树

| ID | 需求 | 详情 |
|----|------|------|
| TSK-01 | WBS 树形结构 | 支持无限层级父子任务，DFS 遍历 |
| TSK-02 | 自动编号 | 按 `plan_code_rule`（默认 `3,2,2`）自动生成 WBS 编码（如 `001.02.03`） |
| TSK-03 | 节点类型 | 计划节点（摘要）/ 任务节点（叶子）/ 里程碑节点 |
| TSK-04 | 创建任务 | 可选父节点；自动继承父节点部门与责任人 |
| TSK-05 | 编辑任务 | 含优先级（1-4）、状态、日期、工期、责任人、部门、前置依赖、里程碑关联 |
| TSK-06 | 删除任务 | 递归删除子任务；自动清理其他任务中的前置引用 |
| TSK-07 | 重新编号 | 支持对整个项目或单个子树重新生成序号 |

### 5.2 前置依赖

| ID | 需求 | 详情 |
|----|------|------|
| TSK-08 | 4 种依赖类型 | FS（完成-开始）、SS（开始-开始）、FF（完成-完成）、SF（开始-完成） |
| TSK-09 | 滞后天数 | LagDays −999～999 |
| TSK-10 | 自动计算日期 | 新增/编辑任务时根据前置约束自动推算计划开始日期 |
| TSK-11 | 自引用防护 | 不可选择自身及子孙节点作为前置 |
| TSK-12 | 可视化 | 甘特图中用箭头连接依赖关系；悬浮显示任务编号和名称；点击可定位 |

### 5.3 父子自动同步

| ID | 需求 | 详情 |
|----|------|------|
| TSK-13 | 父节点日期 | 计划开始 = min(子节点)；计划完成 = max(子节点) |
| TSK-14 | 父节点进度 | 按子节点计划工期加权平均 |
| TSK-15 | 父节点工期 | 直接子节点工期之和 |
| TSK-16 | 父节点状态 | 子节点全部完成→完成，任一有开始→进行中，否则未开始 |

### 5.4 里程碑特殊规则

| ID | 需求 | 详情 |
|----|------|------|
| TSK-17 | 里程碑工期强制为 0 | 计划工期、参考工期、实际工期均强制归零 |
| TSK-18 | 日期同步 | 计划开始 = 计划完成（仅存一个日期）；实际同理 |
| TSK-19 | 进度判断 | 实际完成→进度 100%；否则 0%；无子节点 |

### 5.5 甘特图

| ID | 需求 | 详情 |
|----|------|------|
| TSK-20 | 日/周/月切换 | 支持按天、按周、按月三种时间粒度 |
| TSK-21 | 依赖箭头 | 用 SVG/Canvas 绘制前置依赖箭头 |
| TSK-22 | 今日线 | 红色竖线标记当天位置 |
| TSK-23 | 进度条 | 任务条内部颜色区分进度（0-100%） |
| TSK-24 | 图例 | 状态/today/里程碑/关键路径等颜色图例 |

### 5.6 跨项目任务视图

| ID | 需求 | 详情 |
|----|------|------|
| TSK-25 | 全局任务列表 | 跨项目汇总，多维度筛选（项目、日期、状态、逾期、责任人） |
| TSK-26 | 数据隔离 | **所有用户（含管理员）只能看到自己作为项目成员的项目中的任务** |

### 5.7 我的任务

| ID | 需求 | 详情 |
|----|------|------|
| TSK-27 | 个人待办 | 仅显示当前用户为责任人的进行中项目任务 |
| TSK-28 | 逾期标记 | 未完成 + 计划完成 < 今天 → `IsOverdue = true` |
| TSK-29 | 待办计数 | 统计 Status=0 或 1 的任务数，展示在侧边栏角标 |

### 6.1 文件清单（版本化管理）

| ID | 需求 | 详情 |
|----|------|------|
| FILE-01 | 模板驱动 | 项目文件清单由文件模板导入，含文件名、是否必填、公开/非公开、可见角色 |

### 6.2 文件可见性

| ID | 需求 | 详情 |
|----|------|------|
| FILE-05 | 公开文件 | 拥有 `project:file:view:public` 权限的所有项目成员可见 |

### 6.3 文件状态与预警

| ID | 需求 | 详情 |
|----|------|------|
| FILE-09 | 逾期标记 | 今日 > 计划完成且仍未上传 → `overdue` |
| FILE-10 | 即将到期 | 距离计划完成 ≤ 7 天且未上传 → `expiring` |
| FILE-11 | 正常 | 距离计划完成 > 7 天 → `normal` |

### 6.4 我的文件

| ID | 需求 | 详情 |
|----|------|------|
| FILE-12 | 个人文件列表 | 显示当前用户在激活项目中作为责任人的文件项 |
| FILE-13 | 待传统计 | 统计未上传总数和必填未上传数，展示在侧边栏角标 |

## 8. 计划套装（PlanBundle）

### 8.1 套装概念

将多个**计划模板**打包为一个套装，一键装配生成**完整的项目 WBS 任务树**。

| ID | 需求 | 详情 |
|----|------|------|
| BDL-01 | 套装 CRUD | 名称、描述、包含的模板列表（含排序） |
| BDL-02 | 模板限制 | 仅**计划类型（TemplateType=1）**且 Status=1 的模板可加入 |
| BDL-03 | 自动去重 | 同一模板不可重复加入 |

### 8.2 装配

| ID | 需求 | 详情 |
|----|------|------|
| BDL-04 | 装配到新模板 | 所有模板节点汇入统一根节点"项目"，自动重新编号 |
| BDL-05 | 依赖重映射 | 保留原模板内的依赖关系，跨模板无依赖 |
| BDL-06 | 输出模板 | 装配结果生成一个新的计划模板（编号 `ASM-yyyyMMddHHmmss`） |

## 14. 非功能需求

### 14.1 性能

| ID | 需求 | 详情 |

| NF-07 | 无 SQL 注入 | 全部参数化查询（EF Core） |
| NF-08 | CORS | 开发环境允许跨域 |
