## 4. 各层详细变更

### ① Program.cs — 扩展上传中间件

**文件**：`codes/backend/ProjectManagement.API/Program.cs`
**位置**：第 251–280 行（现有 `upload_filesize` 中间件内）

**改动**：在同一个中间件中增加 `upload_filemaxtime` 读取 + 请求超时设置。

```
现有逻辑：
  ├─ 读取 upload_filesize → 设置 MaxRequestBodySize
  └─ await next()

新增逻辑：
  ├─ 读取 upload_filesize → 设置 MaxRequestBodySize
  ├─ 读取 upload_filemaxtime → 解析秒数（默认 60）
  ├─ 上传类路径 → IHttpRequestTimeoutFeature.SetTimeout()
  └─ await next()
```

**关键代码**：

```csharp
// 在现有 SysParams 查询处追加
var timeoutParam = await db.SysParams
    .FirstOrDefaultAsync(p => p.ParamKey == "upload_filemaxtime");
int timeoutSeconds = 60; // 默认 60 秒
if (timeoutParam != null && int.TryParse(timeoutParam.ParamValue.Trim(), out var sec) && sec > 0)
    timeoutSeconds = sec;
UploadConfig.CurrentTimeoutSeconds = timeoutSeconds;

// 仅对上传类路径设置超时
var path = context.Request.Path.Value ?? "";
if (path.Contains("/upload") || (path.Contains("/files") && context.Request.Method == "POST"))
{
    var timeoutFeature = context.Features.Get<IHttpRequestTimeoutFeature>();
    if (timeoutFeature != null)
        timeoutFeature.SetTimeout(TimeSpan.FromSeconds(timeoutSeconds));
}
```

**`UploadConfig` 静态类新增**（第 349–352 行）：

```csharp
public static class UploadConfig
{
    public static long CurrentLimit { get; set; } = 100L * 1024 * 1024;
    public static int CurrentTimeoutSeconds { get; set; } = 60; // 新增
}
```

### ② ExceptionMiddleware.cs — 超时异常处理

**文件**：`codes/backend/ProjectManagement.API/Middleware/ExceptionMiddleware.cs`
**位置**：第 40–45 行（`ex switch` 表达式）

**改动**：新增超时异常分支：

```csharp
message = ex switch
{
    UnauthorizedAccessException => "没有权限执行此操作",
    var e when e.GetType().Name == "BadHttpRequestException" =>
        $"上传文件大小超过系统限制（最大 {UploadConfig.CurrentLimit / 1024 / 1024} MB），请压缩后重试",
    // ↓ 新增
    var e when (e is OperationCanceledException or TimeoutException 
                || e.GetType().Name == "BadHttpRequestException" 
                && e.Message.Contains("timeout"))
        && context.RequestAborted.IsCancellationRequested =>
        $"文件上传超时（最大 {UploadConfig.CurrentTimeoutSeconds} 秒），请检查网络或压缩文件后重试",
    _ => ex.Message
}
```

### ③ project.ts — 上传 API 超时覆盖

**文件**：`codes/frontend/src/api/project.ts`
**位置**：第 195–209 行（`uploadProjectFileItem` 函数）

**改动**：上传请求覆盖 axios 全局 15s 超时。

```typescript
// 辅助函数（文件内或新建 utils/uploadConfig.ts）
function getUploadTimeout(): number {
  // 从 localStorage 读取系统参数 upload_filemaxtime 缓存值
  const cached = localStorage.getItem('upload_filemaxtime')
  if (cached) {
    const sec = parseInt(cached)
    if (sec > 0) return sec * 1000  // axios timeout 单位毫秒
  }
  return 60000 // 默认 60 秒
}

// uploadProjectFileItem 中
return request.post<...>(url, formData, {
  headers: { 'Content-Type': 'multipart/form-data' },
  timeout: getUploadTimeout(), // 覆盖全局 15s
  onUploadProgress: (e) => { ... }
})
```

> **注意**：`upload_filemaxtime` 的值需要前端缓存。可复用现有获取系统参数的 API，在 `AuthStore` 或页面初始化时拉取并存到 localStorage。
