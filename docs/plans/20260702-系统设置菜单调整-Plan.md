## 3. 实施细节

### 3.1 静态菜单（menu.ts）

新增独立一级菜单：

```typescript
{
  path: '/settings',
  title: '系统设置',
  icon: 'Setting',
  roles: ['admin', 'templateAdmin', 'user'],
  children: [
    { path: '/system/settings', title: '修改密码', icon: 'Lock', roles: ['admin', 'templateAdmin', 'user'] }
  ]
}
```

roles 包含 `admin` / `templateAdmin` / `user`，保证所有登录用户可见。

### 3.2 权限码 Seed（DbInitializer）

| 权限码 | 名称 | 类型 | 父节点 |
|--------|------|------|--------|
| `settings` | 系统设置 | Menu (Type=1) | — |
| `settings:password` | 修改密码 | Menu (Type=1) | settings |

- 权限码自动分配给全部 6 个 RBAC 角色（admin / template_admin / project_admin / project_manager / task_manager / user）
- 已有数据库中存在 Icon 为空或 Path 为空时，用原始 SQL 强制修复

### 3.3 图标渲染修复（SubMenuRenderer.vue）

`el-sub-menu` 展开时 `<el-icon>` 包裹的图标会因 Element Plus 内部处理而丢失。改为直接使用 Vue 的 `<component :is="图标名">` 渲染，绕过该问题。
