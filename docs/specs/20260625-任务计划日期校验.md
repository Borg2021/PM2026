# [任务计划日期校验：计划完成不能早于计划开始]

> 日期：2026-06-25　|　状态：已实现

## 1. 为什么要做（背景）

保存任务计划时发现，任务"通知货架发货"（序号 001.01.03.03.01）的计划完成（2026-08-01）早于计划开始（2026-08-05），逻辑上不合法。前后端均缺少此校验，导致异常数据入库。

## 2. 要达成什么（需求）

- [x] 保护数据完整性：任何写入路径下，`planFinishDate` 都不小于 `planStartDate`
- [x] 行内批量保存时，有问题的任务跳过并提示，不影响其他任务的正常保存

## 3. 怎么算做完（验收标准）

- [x] **后端**：`UpdateProjectTaskHandler` 写入前校验 `PlanFinishDate >= PlanStartDate`，不满足时抛 `InvalidOperationException`
- [x] **后端**：里程碑（`nodeType == 2`）自动同步 `PlanStartDate = PlanFinishDate`
- [x] **前端**：`saveListEdits` 中每个脏行在保存前交叉校验日期，非法行跳过并弹窗警告
- [x] **现有校验保留**：弹窗编辑 `handleSaveTask`（line 1340）和行内单字段编辑 `handleDateChange`（line 1733）原有校验不受影响

## 4. 影响范围

- `ExtendedProjectCommands.cs` — `UpdateProjectTaskHandler`
- `ProjectEditor.vue` — `saveListEdits` 函数

## 5. 实现摘要

### 后端

- **`ExtendedProjectCommands.cs:147-152`**：在 `UpdateProjectTaskHandler` 中，所有字段赋值前插入校验块：
  1. 里程碑节点自动同步 `planStartDate = planFinishDate`
  2. 当 `PlanFinishDate < PlanStartDate` 时抛 `InvalidOperationException`

### 前端

- **`ProjectEditor.vue:1780-1784`**：`saveListEdits` 循环脏行时，在 `applyInlineCalc` 之后、API 调用之前插入交叉校验。非法行通过 `continue` 跳过，不影响其他行保存。
