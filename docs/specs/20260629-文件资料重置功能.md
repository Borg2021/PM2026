# 文件资料重置功能

> 日期：2026-06-29　|　状态：已实现

## 1. 为什么要做（背景）

当前文件资料上传后，状态变为「已上传」，但某些场景下需要让文件项重新回到「待上传」状态：

- 上传了错误的文件，需要责任人重新提交
- 版本说明与实际内容不符，需要重新上传
- 审核后发现文件不合格，需要打回重传

目前管理员只能查看文件状态，无法将已上传的文件项重置为待上传状态（除非手动操作数据库）。

## 2. 要达成什么（需求）

- [ ] 需求点 1 新增后端 API `POST /api/v1/projects/{id}/file-items/{itemId}/reset`，**仅系统管理员**（admin 角色）可调用
- [ ] 需求点 2 重置操作将文件项的 `LatestVersionId` 置空，使状态恢复为「待上传」，**历史版本记录和物理文件保留不删除**
- [ ] 需求点 3 前端「文件资料」tab 的操作列中，对已上传的文件项增加「重置」按钮，**仅管理员可见**
- [ ] 需求点 4 前端「文件管理」全局视图（`/project/files`）同样增加重置按钮，规则一致
- [ ] 需求点 5 点击重置时弹出确认对话框，确认后执行重置并刷新列表
- [ ] 需求点 6 重置操作记录到项目操作日志，格式：「重置文件「{fileName}」状态为待上传」

## 3. 怎么算做完（验收标准）

> 每条标准必须可验证，忌"功能正常"这种模糊描述

- [ ] 场景 A：管理员在项目 → 文件资料 tab → 已上传的文件项操作列出现「重置」按钮 → 点击 → 弹出确认框 → 确认 → 状态变为「待上传」，操作日志中出现重置记录
- [ ] 场景 B：管理员在项目 → 文件资料 tab → 待上传的文件项操作列**不出现**「重置」按钮
- [ ] 场景 C：非管理员（项目经理、普通成员、责任人）在项目 → 文件资料 tab → 任何文件项操作列**不出现**「重置」按钮
- [ ] 场景 D：管理员重置文件后 → 该文件项仍可正常重新上传 → 上传后状态变回「已上传」，版本号在上一个版本号基础上递增
- [ ] 场景 E：管理员重置文件后 → 点击「版本」仍能看到历史版本记录和可下载的历史文件
- [ ] 场景 F：管理员在全局「文件管理」页面 → 已上传的文件项操作列同样出现「重置」按钮 → 行为与项目内一致
- [ ] 场景 G：直接调用 `POST /api/v1/projects/{id}/file-items/{itemId}/reset` → 非 admin 用户返回 403

## 4. 影响范围（初判）

- **涉及层**：API（Controller + 操作日志）、Frontend（两个文件的 Vue 组件）
- **后端文件**：`ProjectController.cs`（新增 reset 端点）
- **前端文件**：
  - `src/views/project/components/ProjectFileTab.vue`（项目内文件资料 tab）
  - `src/views/project/ProjectFiles.vue`（全局文件管理视图）

## 5. 设计要点

### 5.1 权限模型

| 角色 | 能否重置 |
|------|---------|
| 系统管理员（admin） | ✅ 可以 |
| 项目经理 | ❌ 不可 |
| 文件责任人 | ❌ 不可 |
| 项目成员 | ❌ 不可 |

### 5.2 数据操作

```
重置前：ProjectFileItem.LatestVersionId = 5  →  状态「已上传」
重置后：ProjectFileItem.LatestVersionId = null  →  状态「待上传」

保留数据：
- ProjectFileVersion 记录不删除
- ProjectFileVersionFile 记录不删除
- 物理文件不删除
```

### 5.3 操作日志格式

```
日志类型: 文件管理
操作: 重置文件「{fileName}」状态为待上传
项目: {projectName}
操作人: {userName}
```

### 5.4 前端按钮逻辑

```
显示「重置」按钮的条件（AND）：
1. 当前用户是系统管理员（admin 角色）
2. 文件项状态为「已上传」（item.hasUpload === true）

按钮样式：type="warning" size="small" link
按钮文案：「重置」
```

## 6. 实现摘要

| 层 | 文件 | 改动 |
|---|------|------|
| API | ProjectController.cs | 新增 `POST {id}/file-items/{itemId}/reset` 端点，校验 admin 角色，置空 `LatestVersionId`，记录操作日志 |
| API | project.ts | 新增 `resetProjectFileItem(projectId, itemId)` 函数 |
| Frontend | ProjectFileTab.vue | 导入 `resetProjectFileItem`，新增 `isAdmin` 计算属性 + `handleReset` 函数，操作列新增「重置」按钮（仅 admin + 已上传可见） |
| Frontend | ProjectFiles.vue | 导入 `resetProjectFileItem` + `useAuthStore`，新增 `isAdmin` + `handleFileReset` 函数，操作列新增「重置」按钮 |
