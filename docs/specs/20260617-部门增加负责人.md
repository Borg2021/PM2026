# [部门增加负责人]

> 日期：2026-06-17　|　状态：已完成

## 1. 为什么要做（背景）

项目成员所在的部门负责人，以及部门的上级部门的负责人看不到（递归，上级部门的上级部门的上级部门...的负责人)， 如果有人员在项目 成员列表中，则这些负责人可以看到，不需要单独授权。

## 2. 要达成什么（需求）

- [x] 需求点 1 在人员管理 点击某个部门， 在当前部门 的 新建用户 的左侧 ，增加 设置负责人， 如果有部门负责人， 则 在 当前部门: 部门  负责人 某某某
- [x] 需求点 2 设置部门的负责人的选择范围 可以是任何人;
- [x] 需求点 3 部门负责人 可以不是当前部门的成员;
- [x] 需求点 4 部门负责人可以设置多个;

## 3. 怎么算做完（验收标准）

> 每条标准必须可验证，忌"功能正常"这种模糊描述
- [x] 在人员管理页面，选中某部门后，工具栏出现"设置负责人"按钮，可搜索任意用户并设为负责人;
- [x] 负责人设置成功后，工具栏显示「当前部门: XX | 负责人: XX」;
- [x] 部门负责人可以不是该部门成员，选择范围不限于当前部门人员;
- [x] 可为一个部门设置多个负责人，支持多选（`el-select` 多选 + 远程搜索）;

## 4. 影响范围（初判）

- **数据库**：`Departments.LeaderId`（单字段）→ `DepartmentLeaders` 关联表（DepartmentId + UserId，唯一索引防重复），旧数据自动迁移
- **后端 API**：`SystemController` + `CommonController` 的 Department CRUD，请求体 `leaderId` → `leaderIds[]`，响应 `leaderId/leaderName` → `leaders: [{userId, realName}]`
- **权限可见性**：`ProjectDataScopeFilter.Apply`（列表 Union）、`MatchesProjectAsync`（单项目）、`CanAccessProjectAsync`（详情）均改为查询 `DepartmentLeaders` 表
- **前端**：`UserManagement.vue` 工具栏 Tag 列表 + `el-select` 多选对话框

## 5. 实现摘要

### 数据库 / 后端

- **新增 `DepartmentLeader` 实体**：Department → DepartmentLeader（1:N），替代原 `Department.LeaderId`/`Leader` 单字段。
- **数据库迁移**：建 `DepartmentLeaders` 表（3 个索引 + 唯一约束），自动将旧 `Departments.LeaderId` 数据迁移到新表。
- **Department API**：`GET` 返回 `leaders: [{userId, realName}]`；`POST/PUT` 接受 `leaderIds: number[]`（先删后加）。
- **用户搜索 API**：`GET /api/v1/users/search` 新增返回 `departmentName`。
- **项目可见性**：`ProjectDataScopeResolver.GetLeaderDeptTreeIdsAsync` 改为查 `DepartmentLeaders` 表递归收集部门树 → `ProjectListScopeFilter.LeaderDeptIds` → `ProjectDataScopeFilter.Apply` 用 `Union` 合并可见项目。`CanAccessProjectAsync` 早期检查部门负责人身份。

### 前端

- **类型定义**：`DepartmentItem` 从 `leaderId/leaderName` 改为 `leaders: {userId, realName}[]`。
- **API 层**：`createDepartment`/`updateDepartment` 参数从 `leaderId` 改为 `leaderIds[]`，新增 `searchUsers`。
- **UserManagement.vue**：工具栏显示多个 `el-tag`，对话框改为 `el-select` 多选 + `remote` 远程搜索 + `reserve-keyword`，打开时预填当前负责人避免显示编号。
