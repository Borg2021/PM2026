# [部门增加负责人]

> 日期：2026-06-17　|　状态：已完成

## 1. 为什么要做（背景）

人员管理、部门管理、职能管理 的 页面滚动条不协调

## 2. 要达成什么（需求）

- [x] 需求点 1 在人员管理界面 左侧的部门结构 的 高度 和 右侧 的部门人员列表的高度要一致， 如果超出页面高度，不要整体页面滚动，要 部门人员列表内部滚动条， 现在的模式如果滚动条下拉，部门结构也会随着一起鼓滚动；
- [x] 需求点 2 在部门管理页面 滚动条在部门列表内部，不要整个页面滚动；
- [x] 需求点 3 在职能管理页面 滚动条在职能列表内部，不要整个页面滚动；

## 3. 怎么算做完（验收标准）

> 每条标准必须可验证，忌"功能正常"这种模糊描述


## 4. 影响范围（初判）

三个页面的 `<style scoped>` CSS + 少量模板结构，不改业务逻辑、不改后端。

## 5. 实现摘要

### 前端

- **UserManagement.vue**：`.page-container` 改为 `height: 100%` + flex column + `overflow: hidden`，`.dept-user-layout` 占满剩余空间（`flex: 1`），左侧 `.dept-tree-card` 和右侧 `.user-content` / `.table-card` 各自内部滚动。左右高度一致。
- **DepartmentManagement.vue**：`.page-container` 同上，表格外层加 `.table-wrapper`（`flex: 1` + `overflow: auto`），实现表格区域内部滚动。
- **FunctionManagement.vue**：同 DepartmentManagement，表格区域内部滚动。
