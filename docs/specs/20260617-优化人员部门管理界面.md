# 优化人员/部门/职能管理界面滚动

> 日期：2026-06-17　|　状态：已完成

## 1. 为什么要做（背景）

人员管理、部门管理、职能管理 的 页面滚动条不协调

## 2. 要达成什么（需求）

- [x] 需求点 1 在人员管理界面 左侧的部门结构 的高度 和 右侧 的部门人员列表的高度要一致， 如果超出页面高度，不要整体页面滚动，要 部门人员列表内部滚动条， 现在的模式如果滚动条下拉，部门结构也会随着一起鼓滚动；
- [x] 需求点 2 在部门管理页面 滚动条在部门列表内部，不要整个页面滚动；
- [x] 需求点 3 在职能管理页面 滚动条在职能列表内部，不要整个页面滚动；

## 3. 怎么算做完（验收标准）

- [x] 人员管理：左侧部门树与右侧人员列表高度一致，右侧内部滚动
- [x] 人员管理：人员列表表头固定不动，仅数据行内部滚动，分页栏固定在底部
- [x] 部门管理：部门列表表头固定不动，仅数据行内部滚动
- [x] 职能管理：职能列表表头固定不动，仅数据行内部滚动

## 4. 影响范围（初判）

三个页面的 `<style scoped>` CSS + 少量模板结构，不改业务逻辑、不改后端。

## 5. 实现摘要

### 前端

- **UserManagement.vue**：`.page-container` 改为 `height: 100%` + flex column + `overflow: hidden`，`.dept-user-layout` 占满剩余空间（`flex: 1`），左侧 `.dept-tree-card` 和右侧 `.user-content` / `.table-card` 各自内部滚动。左右高度一致。后续迭代：用 `:deep()` 穿透 `el-table` 内部结构，`el-table__body-wrapper` → `flex: 1; overflow-y: auto` 仅数据行滚动，`el-table__header-wrapper` → `flex-shrink: 0` 表头固定，分页栏固定在 `.table-card` 底部。
- **DepartmentManagement.vue**：`.page-container` 同上，去掉外层 `table-wrapper`，改为用 `:deep()` 让 `el-table` 撑满剩余空间，表头固定 + body 滚动。
- **FunctionManagement.vue**：同 DepartmentManagement，表头固定 + body 滚动。
