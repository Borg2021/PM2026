# [文件资料上传多个文件]

> 日期：2026-06-17　|　状态：已完成

## 1. 为什么要做（背景）

用户在 上传文件 时，当前只能上传单个文件，有时一个文件资料有多个文件。

## 2. 要达成什么（需求）

- [x] 需求点 1 上传文件时，可以选择多个文件上传；
- [x] 需求点 2 文件操作点击 下载 ，弹出文件列表，点击某个文件进行下载。
- [x] 需求点 3 文件操作点击 版本 ， 弹出的界面中，在版本后面增加1列，列出这个版本可以下载的文件， 点击某个文件进行下载， 原操作列 取消；

## 3. 怎么算做完（验收标准）

> 每条标准必须可验证，忌"功能正常"这种模糊描述
- [x] 在文件管理列表中， 点击 上传 或者 重传， 可以选择多个文件上传;
- [x] 上传完成后， 点击下载， 弹出框，列出文件列表 ，点击某个文件进行下载; 
- [x] 同需求点3;

## 5. 实现摘要

### 数据库 / 后端

- **新增 `ProjectFileVersionFile` 实体**：`ProjectFileVersion` → `ProjectFileVersionFile`（1:N），每个版本可关联多个物理文件。移除了 `ProjectFileVersion` 上的旧单文件字段（`FilePath`, `OriginalFileName`, `FileSize`, `FileExt`）。
- **数据库迁移 SQL**：在 `Program.cs` 中内联创建 `ProjectFileVersionFiles` 表 + 索引，并将旧 `ProjectFileVersions` 中的单文件数据自动迁移到新表。
- **上传 API** (`POST .../upload`)：`IFormFile` 参数改为 `List<IFormFile>`，遍历保存每个文件到 `ProjectFileVersionFiles` 表，物理文件存入 `ProjectFiles/{projectId}/{guid}.{ext}`。
- **下载 API** (`GET .../download`)：新增 `fileId` 查询参数，指定时直接下载对应 `ProjectFileVersionFile`；未指定时兼容旧行为（取最新版本第一个文件）。
- **版本历史 API** (`GET .../versions`)：每个版本返回 `files` 数组（`id`, `originalFileName`, `fileSize`, `fileExt`）。
- **删除 API** (`DELETE .../file-items/{itemId}`)：级联清理 `ProjectFileVersionFiles` 中的物理文件。
- **LatestVersion DTO**：`GetFileItems` / `GetMyFiles` 返回的 `latestVersion` 包含 `files` 数组。

### 前端

- **类型定义**：新增 `ProjectFileVersionFile` 接口，`ProjectFileVersion` 增加 `files` 字段，`MyFileItem.latestVersion` 同步更新。
- **API 层**：`uploadProjectFileItem` 改为接受 `File[]`，`getFileDownloadUrl` 增加 `fileId` 参数。
- **ProjectFileTab.vue**：上传改为 `<input multiple>` 多文件选择，下载按钮改为弹出文件列表弹窗，版本历史对话框移除操作列、新增"文件"列展示各版本文件。
- **ProjectFiles.vue**（全局文件管理视图）：同样改造为多文件上传 + 下载弹窗 + 版本对话框文件列。
- **修复**：`window.open` 内联调用改为独立 handler 函数，移除 `as any` 类型断言。
