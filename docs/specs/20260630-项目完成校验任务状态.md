# 确认完成时校验计划任务是否全部完成

> 日期：2026-06-30　|　状态：已实现

## 1. 为什么要做（背景）

当前点击"确认完成"按钮没有任何前置校验，即使项目下还有大量未完成的任务，也能直接将项目标记为已完成。用户误操作后无法自行恢复（此前已完成状态没有操作按钮，已修复）。

需要增加完成前的任务状态校验，防止项目在有未完成任务的情况下被误标记为完成。

### 1.1 现状分析

| 组件 | 现状 |
|------|------|
| 后端 `Complete` 接口 | 直接改状态为 2，无任何前置校验 |
| 前端 `handleComplete` | 仅弹出确认对话框，无额外检查 |
| 系统参数 | 无 `is_project_force_finish` 参数 |

### 1.2 设计决策

引入系统参数 `is_project_force_finish` 控制行为：

| 参数值 | 行为 |
|--------|------|
| `"1"` | 强制完成，跳过任务状态检查 |
| `"0"` 或不存在 | 检查所有计划任务是否已完成，有未完成则拒绝 |

## 2. 要达成什么（需求）

- [x] 需求点 1 系统参数表新增 `is_project_force_finish` 参数（key=is_project_force_finish, value=0），管理员可在系统参数管理页面修改
- [x] 需求点 2 点击"确认完成"时，后端先读取 `is_project_force_finish` 参数：
  - 值为 `"1"` → 跳过检查，直接完成
  - 值为 `"0"` 或不存在 → 检查项目下所有计划任务状态
- [x] 需求点 3 任务状态检查：查询该项目所有 `ProjectTask`，若存在 `Status != 2`（非"已完成"）的任务，返回错误信息"该项目有未完成的任务，不能标记为完成"
- [x] 需求点 4 前端已自动处理 `code !== 0` 的响应（显示 `ElMessage.error`），无需额外改动

## 3. 怎么算做完（验收标准）

- [x] 场景 A：`is_project_force_finish` = `"1"` → 项目有未完成任务 → 点击确认完成 → 成功，状态变为已完成
- [x] 场景 B：`is_project_force_finish` = `"0"` → 项目所有任务 Status=2 → 点击确认完成 → 成功
- [x] 场景 C：`is_project_force_finish` = `"0"` → 项目有任务 Status≠2 → 点击确认完成 → 前端弹出错误"该项目有未完成的任务，不能标记为完成"
- [x] 场景 D：`is_project_force_finish` 参数不存在 → 项目有未完成任务 → 行为同场景 C（默认拒绝）
- [x] 场景 E：项目无任何任务 → 点击确认完成 → 成功（无任务视为全部完成）
- [x] 场景 F：系统参数管理页面可以看到并修改 `is_project_force_finish` 参数

## 4. 影响范围（初判）

- **涉及层**：Backend only（前端无需改动，响应拦截器已处理错误提示）
- **后端文件**：`ProjectController.cs` — `Complete` 方法
- **数据库**：`SysParams` 表需插入一行 `is_project_force_finish` 初始值

## 5. 设计要点

### 5.1 后端 Complete 方法改造

```
[HttpPost("{id:long}/complete")]
async Task<IActionResult> Complete(long id)
  1. 权限校验（不变）
  2. 读取 SysParams 中的 is_project_force_finish
  3. 如果值 != "1"：
     a. 查询该项目下 Status != 2 的任务数量
     b. 如果 count > 0 → 返回 { code: 400, message: "该项目有未完成的任务，不能标记为完成" }
  4. 执行状态变更（不变）
```

### 5.2 数据库初始化

在 `DbInitializer.EnsureSeedAsync` 中自动插入初始参数（应用启动时检查）：

```csharp
if (!db.SysParams.Any(p => p.ParamKey == "is_project_force_finish"))
{
    db.SysParams.Add(new SysParam { ParamKey = "is_project_force_finish", ParamValue = "0", Description = "..." });
    await db.SaveChangesAsync();
}
```

如果已存在则跳过，管理员也可通过"系统参数管理"页面手动添加和修改。
