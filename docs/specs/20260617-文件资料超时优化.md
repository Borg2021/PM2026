# [文件资料上传超时优化]

> 日期：2026-06-17　|　状态：已实现

## 1. 为什么要做（背景）

用户在上传大文件时，超过默认的15秒，导致超时，不能完成文件上传。

## 2. 要达成什么（需求）

- [x] 需求点 1 在系统参数 中添加 upload_filemaxtime ，单位秒
- [x] 需求点 2 上传文件时，如果超过系统参数 upload_filemaxtime 设置的时间，则报错，如果系统没有设置 upload_filemaxtime 这个参数，则默认 60秒。

## 3. 怎么算做完（验收标准）

> 每条标准必须可验证，忌"功能正常"这种模糊描述

- [x] 场景 A：系统参数  upload_filemaxtime 设置 5 ， 用户在项目管理的文件资料 列表中 上传某个文件，150MB 的文 → 预期结果 系统报错，提示上传文件超时
- [x] 场景 B：系统参数  upload_filemaxtime 设置 120 ， 用户在项目管理的文件资料 列表中 上传某个文件，150MB 的文 → 预期结果 系统上传文件正常

## 4. 影响范围（初判）

- 涉及层：API（中间件 + 异常处理 + Controller）、Frontend（API 层 + Store）
- 后端文件：Program.cs、ExceptionMiddleware.cs、ProjectController.cs
- 前端文件：src/api/project.ts、src/api/auth.ts、src/store/auth.ts

## 5. 实现摘要

| 层 | 文件 | 改动 |
|---|------|------|
| API | Program.cs | 中间件新增 upload_filemaxtime 读取 + 请求级超时（CancellationToken 链接） |
| API | ExceptionMiddleware.cs | 新增 OperationCanceledException / TimeoutException 中文提示 |
| API | ProjectController.cs | 3 个上传方法新增 CancellationToken 参数 |
| Frontend | src/api/project.ts | getUploadTimeout() 从 localStorage 读超时 + uploadProjectFileItem 覆盖 axios timeout |
| Frontend | src/api/auth.ts | fetchUploadConfig() 拉取系统参数并缓存 |
| Frontend | src/store/auth.ts | 登录后自动调用 fetchUploadConfig() |
