# 项目编辑页 Tab 首次加载等待效果

> 日期：2026-06-30　|　状态：已实现

## 1. 为什么要做（背景）

用户打开项目编辑页时，各个 tab 在数据加载完成前直接显示空表格和"暂无数据"文案，给用户造成"这个页面没有内容"的错觉，体验不佳。

### 1.1 现状分析

| Tab | 数据来源 | 加载时机 | 当前 loading 状态 |
|-----|---------|---------|------------------|
| 基本信息 | `loadDetail()` | onMounted | `v-loading="loading"` 覆盖整个 tabs 容器 ✅ |
| 成员列表 | `loadDetail()` | onMounted | 同上 ✅ |
| 文件资料 | 组件内 API | 首次切到该 tab | 组件内部有独立 `loading` ✅ |
| 任务计划 | `loadTasks()` | 首次切到该 tab | **无 loading 指示** ❌ |
| 里程碑 | `loadTasks()`（复用） | 首次切到该 tab | **无 loading 指示** ❌ |
| 考核任务 | `loadTasks()`（复用） | 首次切到该 tab | **无 loading 指示** ❌ |
| 任务列表 | `loadTasks()`（复用） | 首次切到该 tab | **无 loading 指示** ❌ |
| 甘特图 | `loadGanttData()` | 首次切到该 tab | `ganttLoading` ✅ |

### 1.2 问题根源

1. **全局 `loading` 变量**只覆盖"基本信息"表单加载阶段，一旦基本信息加载完毕就设为 `false`，此时其他 tab 的数据可能还没加载
2. `loadTasks()` 被调用时没有设置任何 loading 标记，用户点击任务计划/里程碑/考核任务/任务列表 tab 时直接看到空表格和"暂无数据"文案，等待 API 返回后才突然刷出数据
3. 甘特图已有 `ganttLoading`，但其他依赖 `loadTasks()` 的 tab 缺少同样的机制

## 2. 要达成什么（需求）

- [ ] 需求点 1 任务计划、里程碑、考核任务、任务列表 tab 首次加载时显示 loading 遮罩（`v-loading`），避免用户看到"暂无数据"空状态
- [ ] 需求点 2 loading 文案统一为"加载中..."，与甘特图一致
- [ ] 需求点 3 如果用户先切到任务计划 tab 触发数据加载，此时切换到里程碑等 tab 应同步显示加载状态（共享 `tasks.value` 数据源）
- [ ] 需求点 4 数据加载完成后，如果确实没有数据，才显示原有空状态文案（如"暂无任务数据"、"暂无里程碑数据"等）
- [ ] 需求点 5 非首次加载（数据已缓存）时不再显示 loading，直接展示数据

## 4. 影响范围（初判）

- **涉及层**：Frontend only（纯前端体验优化，无需后端改动）
- **前端文件**：`ProjectEditor.vue`

## 5. 设计要点

### 5.1 新增 loading 状态

```typescript
const tasksLoading = ref(false)  // 任务计划/里程碑/考核任务/任务列表共用
```

`loadTasks()` 内部设置该状态：

```typescript
async function loadTasks() {
  if (!projectId.value) return
  tasksLoading.value = true
  try {
    const res = await getProjectTasks(projectId.value)
    tasks.value = res.data
    syncParentPlanDates()
  } finally {
    tasksLoading.value = false
  }
}
```

### 5.2 各 tab 使用方式

| Tab | loading 绑定 | 说明 |
|-----|------------|------|
| 任务计划 | `v-loading="tasksLoading"` | 放在表格外层容器 |
| 里程碑 | `v-loading="tasksLoading"` | 表格区域 |
| 考核任务 | `v-loading="tasksLoading"` | 表格区域 |
| 任务列表 | `v-loading="tasksLoading"` | 看板区域 |
| 甘特图 | 已有 `ganttLoading` | 不变 |

### 5.3 loading 指令位置

每个 tab 的内容区域（card/table wrapper）加上 `v-loading`，不覆盖 tab 切换头部。

示例（任务计划）：

```html
<el-tab-pane label="任务计划" name="tasks">
  <div v-loading="tasksLoading" element-loading-text="加载中...">
    <!-- 工具栏 + 表格 -->
  </div>
</el-tab-pane>
```

### 5.4 避免闪烁

- 如果 `tasks.value.length > 0` 说明数据已缓存，`loadTasks()` 不会再发起请求（`onTabChange` 中有 `tasks.value.length === 0` 判断）
- 因此 `tasksLoading` 只在首次加载时为 `true`，后续切换不会触发 loading
