# [前置任务跳转高亮：固定列背景不显示]

> 日期：2026-06-29　|　状态：✅ 已修复（最终方案：JS inline `!important`）

## 1. 问题描述

点击前置任务序号跳转时，目标行会闪烁高亮背景。但高亮只在**滚动列**显示，**固定列（「操作」按钮列、"序号"列、"任务名称"列）** 没有背景色。

### 影响页面

| 页面 | 路由 | 组件 | 高亮色 |
|------|------|------|--------|
| 任务管理 | `#/project/tasks` | `TaskManagement.vue` | 蓝色 `rgba(64,158,255,0.25)` |
| 项目查看-任务计划 | `#/project/view/:id` | `ProjectEditor.vue` | 粉色 `#fce4ec` |

## 2. 根因

### 2.1 TaskManagement.vue — Element Plus `fixed="right"` 独立 DOM 树

Element Plus 固定列在**独立的 `<table>`**（`.el-table__fixed-right` / `.el-table__fixed-left`）中渲染，与主表格的 `<tr>` 不在同一 DOM 树。原先只标记了主表格 `<tr>` 内的 `<td>`，固定列表格的 `<td>` 完全未被触及。

### 2.2 ProjectEditor.vue — CSS sticky `!important` 不可被样式表覆盖

该页面用 **CSS sticky**（非 Element Plus `fixed="right"`）实现粘性列：

```css
:deep(.col-sticky-right) { background: #fff !important; }
:deep(td.col-index)     { background: #fff !important; }
:deep(td.col-taskname)  { background: #fff !important; }
```

任何样式表 CSS（无论有无 `!important`、无论全局/scoped）都**无法击败 scoped 编译后的 `[data-v-xxx] td.col-sticky-right` + `!important`**，因为 CSS 层叠中「同权重 `!important`」由特异度决定，scoped 自动获得属性选择器加成。

### 2.3 第一次尝试失败：CSS `td[data-flash]` + `!important`

全局 `td[data-flash] { background: ... !important }`（特异度 `0,0,1,1`）被 scoped `:deep(td.col-sticky-right)` 编译后的 `[data-v-xxx] td.col-sticky-right { background: #fff !important }`（特异度 `0,0,2,1`）击败。

## 3. 修复方案（最终）

使用 JS **`element.style.setProperty('background', color, 'important')`** — 第三个参数 `'important'` 生成 **inline `!important`**，CSS 优先级高于**任何** stylesheet `!important`（无论 scoped、特异度高低）。

### 3.1 ProjectEditor.vue — `scrollToTaskRow` 函数

```js
if (_ft) clearTimeout(_ft)
// 先清除所有旧高亮
el.querySelectorAll('td[data-flash]').forEach(td => {
  td.style.removeProperty('background')
  td.removeAttribute('data-flash')
})
flashId.value = taskId
const HIGHLIGHT = '#fce4ec'
targetRow.querySelectorAll('td').forEach(td => {
  td.setAttribute('data-flash', '')
  td.style.setProperty('background', HIGHLIGHT, 'important')  // 第三参数 'important' = inline !important
})
_ft = window.setTimeout(() => {
  flashId.value = null
  el.querySelectorAll('td[data-flash]').forEach(td => {
    td.style.removeProperty('background')
    td.removeAttribute('data-flash')
  })
}, 3000)
```

### 3.2 TaskManagement.vue — `scrollToTask` 函数

```js
flashingTaskId.value = taskId
// 清除旧高亮
el.querySelectorAll('td[data-flash]').forEach(td => {
  td.style.removeProperty('background')
  td.removeAttribute('data-flash')
})
const HIGHLIGHT = 'rgba(64, 158, 255, 0.25)'
// 主表格行（滚动列）
targetRow.querySelectorAll('td').forEach(td => {
  td.setAttribute('data-flash', '')
  td.style.setProperty('background', HIGHLIGHT, 'important')
})
// 固定列表格中对应行（独立 DOM 树）
const fixedWrappers = el.querySelectorAll('.el-table__fixed-right, .el-table__fixed-left')
fixedWrappers.forEach(w => {
  const fixedRows = w.querySelectorAll('tr.el-table__row')
  if (fixedRows[pageIdx]) {
    fixedRows[pageIdx].querySelectorAll('td').forEach(td => {
      td.setAttribute('data-flash', '')
      td.style.setProperty('background', HIGHLIGHT, 'important')
    })
  }
})
setTimeout(() => {
  flashingTaskId.value = null
  el.querySelectorAll('td[data-flash]').forEach(td => {
    td.style.removeProperty('background')
    td.removeAttribute('data-flash')
  })
}, 3000)
```

### 3.3 全局 CSS（辅助）

`data-flash` 属性选择器作为 HTML 标记保留，用于 `.cell` 和伪元素透明化：

```css
td[data-flash] .cell { background: transparent !important; }
td[data-flash]::before,
td[data-flash]::after { background: transparent !important; }
```

注意：背景色本身由 JS inline `setProperty` 负责，CSS 中的 `background` 声明可保留但非必须（inline `!important` 优先级最高）。

### 3.4 CSS 优先级层级

```
inline !important          ← JS setProperty(prop, val, 'important')
  ↑ 最高
scoped !important          ← :deep(td.col-sticky-right) { background: #fff !important }
  ↑
全局 !important            ← td[data-flash] { background: ... !important }
  ↑
scoped 普通
  ↑
全局 普通
```

### 3.5 清理机制

- 每次设新高亮前，`el.querySelectorAll('td[data-flash]')` 清除全局所有旧标记（防止连续快速点击残留）
- 3 秒超时后，`style.removeProperty('background')` 恢复 inline style，`removeAttribute('data-flash')` 清除标记
