# 项目编辑页新增「考核任务」Tab

> 日期：2026-06-30　|　状态：已实现

## 1. 为什么要做（背景）

当前项目编辑页面中，「里程碑」tab 展示了任务计划中所有 `nodeType=2`（里程碑类型）的任务节点。用户需要类似的独立视图，快速查看所有任务类别为「考核任务」的任务节点，便于集中管理和追踪考核项。

## 2. 要达成什么（需求）

- [ ] 需求点 1 在「里程碑」tab 旁边新增「考核任务」tab，名称固定为"考核任务"，复用任务计划数据无需额外接口
- [ ] 需求点 2 考核任务 tab 展示任务计划中所有 `taskCategory` 等于「考核任务」对应字典编码的任务节点（含父节点和叶子节点）
- [ ] 需求点 3 考核任务列表展示字段：序号、任务编号、任务名称、状态、计划开始、计划完成、实际开始、实际完成、进度、责任人、操作（查看/编辑）
- [ ] 需求点 4 列表按任务编号排序，支持查看和编辑操作（与里程碑 tab 行为一致）
- [ ] 需求点 5 当 `task_category` 字典中没有「考核任务」时，列表为空并显示"暂无考核任务数据"

## 3. 怎么算做完（验收标准）

- [ ] 场景 A：项目编辑页 → 点击「考核任务」tab → 列出所有任务类别为「考核任务」的任务，按任务编号排序
- [ ] 场景 B：项目中没有任务类别为「考核任务」的任务 → 考核任务 tab 显示空状态"暂无考核任务数据"
- [ ] 场景 C：列表中点击「查看」→ 弹出任务详情对话框
- [ ] 场景 D：非只读模式下，列表中点击「编辑」→ 弹出任务编辑对话框
- [ ] 场景 E：「考核任务」tab 位于「里程碑」tab 右侧
- [ ] 场景 F：`task_category` 字典中不存在「考核任务」字典项 → tab 仍可见，列表为空

## 4. 影响范围（初判）

- **涉及层**：Frontend only（纯前端筛选展示，无需后端改动）
- **前端文件**：`ProjectEditor.vue`（新增 computed + tab pane）

## 5. 设计要点

### 5.1 数据获取

复用已有 `tasks.value`（任务计划数据），前端按 `taskCategory` 筛选。

由于「考核任务」是字典项（非固定编码），通过字典映射找到对应的 `dictCode`：

```typescript
const assessmentTaskCode = computed(() => {
  return dictMap.value['task_category']?.find(d => d.dictLabel === '考核任务')?.dictCode ?? null
})

const assessmentTasks = computed(() =>
  tasks.value
    .filter(t => assessmentTaskCode.value && t.taskCategory === assessmentTaskCode.value)
    .sort((a, b) => (a.taskNo || '').localeCompare(b.taskNo || '', undefined, { numeric: true }))
)
```

### 5.2 Tab 位置

```
... | 任务计划 | 里程碑 | 考核任务(新) | 甘特图 | 任务列表 | ...
```

固定的"考核任务"，不随字典变化动态改变。
