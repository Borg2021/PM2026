# 模板系统

## 4 种模板类型
| 类型 | 枚举值 | 用途 |
|-----|--------|------|
| 计划模板 | `Plan = 1` | WBS 树结构（节点 + 依赖 + 工期） |
| 里程碑模板 | `Milestone = 2` | 预定义里程碑节点 |
| 成员模板 | `Member = 3` | 预定义项目角色分配 |
| 文件模板 | `File = 4` | 预定义文件提交要求 |

## PlanBundle（模板套装）
- 将 4 种模板按顺序组合成一个套装
- 装配（Assemble）到项目时：计划节点 → 任务树，里程碑 → 项目里程碑，成员 → 项目成员，文件 → 文件清单
- 一个模板可以被多个套装引用

## 设计意图
- 不同类型项目（工程/销售/服务）可以用不同套装
- 新建项目时可一键装配套装，减少重复配置
- 模板独立管理，修改模板不影响已装配的项目

**Why:** 模板系统解决了"每个项目都要手动建 WBS、配角色、定义文件清单"的重复劳动问题。参见 [[clean-architecture-layers]]。
